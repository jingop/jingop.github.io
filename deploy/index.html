jingop.github.io

jingop 是你github用户名

# 发布到 github

创建一个文件 deploy.sh

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.github.io
 git push -f git@github.com:jingop/jingop.github.io.git master

# 如果发布到 https://<USERNAME>.github.io/<REPO>
# git push -f git@github.com:<USERNAME>/<REPO>.git master:gh-pages

cd -
  • 注意
 git push -f git@github.com:jingop/jingop.github.io.git master
 
 第一个 jingop 是你的github用户名
 第二个 jingop.github.io 是你创建的仓库名  开头 jingop(github用户名)必须写

# 配置 package.json 的命令

"scripts": {
	"deploy": "bash deploy.sh"
},

# 发布时的命令

使用 git Bash Here 运行命令

npm run deploy
